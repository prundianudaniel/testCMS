Tool–Refactor–Test ist mein pragmatischer Dreischritt, wenn Code alt wird, aber laufen muss. Zuerst Tool: Ich richte ein kleines Hilfswerkzeug ein, das mir Sichtbarkeit gibt – meist ein Linter- oder Coverage-Report. Ohne Messpunkt refactore ich sonst nach Bauchgefühl. Dann Refactor: Ich ändere nur eine Sache pro Durchgang, z.B. eine Funktion entkoppeln oder Namen präzisieren. Danach Test: Ich schreibe oder ergänze genau den Test, der das neue Verhalten absichert.

Mini-Beispiel: Eine KMU-App hatte eine „God“-Methode für Rechnungen. Tool zeigte 0% Coverage. Ich schnitt die Berechnung heraus, refactorte in ein Modul, schrieb zwei Tests für Rabatte – Bug weg, Vertrauen hoch.
